## 〇、术语表（先定义，后引用）

| 术语 | 含义 |
| --- | --- |
| **点亮（一种味道）** | 完成一道家味记忆的"提交 + 讲述"闭环，使其正式成为味道桌上一道已点亮的吃食 |
| **味道桌** | 展示家族所有已点亮吃食的主页面 |
| **还有人在等** | 一道尚未被点亮的家味，等待某位家人来讲，是一种"待办邀请"状态 |
| **抛个钩子** | 由某位家人主动发起"邀请家人来讲某道味道"的动作，本质是创建一条新的"还有人在等" |
| **时间胶囊** | 将一段故事封存到指定日期才可开启的特殊记忆形态 |
| **家谱成员** | 家族家谱树中已录入的所有人物 |

## 一、核心机制：点亮味道 / 抛个钩子 / 还有人在等

### 1.1 "点亮味道"的判定规则（提-讲闭环）
- **判定条件**：只要完成"提出一道味道 → 把它讲述出来"的完整闭环，即视为"点亮"，**不限定是谁提出的、也不限定是不是同一个人讲的**。
  - A 提出后自己讲 → 算点亮 ✅
  - A 提出，B 响应来讲 → 算点亮 ✅
- **核心要求**：闭环达成即可，不绑定具体人物。

### 1.2 "还有人在等" 的来源
- **人工添加**：用户手动创建一条"还有人在等"。
- **AI 自动检测生成**：AI 在对话过程中自动识别并生成。

### 1.3 "抛个钩子" 的本质
- "抛个钩子" = 创建一条新的"还有人在等"。

### 1.4 "抛个钩子" → 指定接收者
- 在"指定给谁讲"处，显示**家族家谱的所有成员**。
- 点击"转给家人"：
  - 支持**单选**关系·姓名。
  - 选中后，自动向所选人物发送通知提醒，提醒类型为"**被点名**"。
- 支持多选（见 1.5）。

### 1.5 指定接收者 · 多选与全选联动
- "指定给谁讲"支持**多选**家谱成员，采用统一的【全选联动规则】（定义见 §1.7）。

### 1.6 "抛个钩子" → 抛出后的完整流程
1. 点击"抛出"后，**自动发送提醒**给选中的家谱成员。
2. 对**晚一步**点击"点亮这道味道"的家谱成员：
   - 弹出**悬浮文字提醒**：「这道家味已被点亮，你生成的记忆已自动补述」。
3. 提醒出现后 **3 秒**自动跳转至"已点亮味道页面"。
4. 该成员 AI 提取的记忆**自动成为补述气泡**的内容。

### 1.7 ★通用规则：全选联动（多处复用）
> 本规则在以下场景统一生效：§1.5"指定给谁讲"、§5.3 时间胶囊"送给谁"。

- 支持**多选**家谱成员。
- **全选按钮双向同步**：
  - 点击"全选" → 自动选中**所有**家谱成员。
  - 取消"全选" → 自动取消选中**所有**家谱成员。
  - 当用户手动选中**全部**家谱成员时 → "全选"按钮**自动点亮**。
  - 当用户取消至**未全选**时 → "全选"按钮**自动取消点亮**。

## 二、AI 交互规范

### 2.1 需要调用 AI 的场景（共 4 类）

| 场景 | 触发时机 | AI 任务 |
| --- | --- | --- |
| **AI 对话 · 点亮前** | 用户进入讲述但尚未点亮 | 必须提取**年份、地点**等信息，生成记忆卡内容 |
| **AI 对话 · 点亮后** | 已点亮后继续讲述 | 读取原有卡片记忆，继续引导提取信息，生成新记忆卡片内容 |
| **人物详情页** | 打开人物详情 | 生成该人物的**个人简介** |
| **每周回顾** | 用户开启该功能后 | 统计一周内家族动态，发送一条通知提醒 |
| **时间胶囊 · 编辑** | 点击"编辑"让 AI 修改时 | AI 自动调取原有卡片内容，继续引导细化，生成新内容 |

### 2.2 AI 对话初始状态（3 种）

| 进入方式 | 是否携带原有记忆 | 是否显示"正在讲 XX（某种味道）"提示 |
| --- | --- | --- |
| ① 点击"讲一种味道" | 否（空白对话） | 否 |
| ② 受"抛个钩子"邀请进入 | 否（空白对话） | 是 |
| ③ 已点亮卡片 → "编辑" → "让 AI 继续补充细节" | 是（携带原卡片记忆） | 是 |

### 2.3 草稿自动保存
- **触发条件**：进入 AI 对话后，只要**未点击"点亮这种味道"** 且 **未点击"删除对话记录"**。
- **结果**：该条记忆自动保存到**草稿箱**。
- **适用范围**：不区分来源 —— 无论是自己创建的，还是受钩子邀请来讲的，规则一致。

### 2.4 AI 默认值规则
- **图片**：用户未上传图片时，记忆卡自动配上**系统默认图**。
- **菜名**：AI 无法从用户记忆中提取出菜名时，默认文字显示为"**未知**"。
- **整理成一段故事 · 缺失信息兜底**：Agent 对话「整理成一段故事」后，对于 AI 没有读取到的信息，在故事生成页（P31）和记忆瞬间页（P11）的对应字段：
  - **年代**：默认填入 **1900**
  - **其余字段**（菜名、地点、正文、图片等）：均显示 **"未知"**

### 2.5 AI 标签生成规则
- **标签数量**：所有「#标签」处由 AI 根据记忆正文内容对应生成 **4-5 个**。
- **动态更新**：标签随记忆正文内容更新而**同步重新生成**。当用户在编辑记忆页修改正文并保存后，AI 自动基于新正文重新提取标签。
- **应用位置**：故事生成页（P31）、记忆瞬间页（P11）、编辑记忆页（P_EditMemory）的标签展示区。

## 三、家族与家谱

### 3.1 个人资料编辑流程

**编辑自己的资料**（从自己头像进入）：
```
头像-姓名 → 性别 → 与谁是什么关系 → 故乡/籍贯 → 代表味道（选填） → 生日
```
- **不需要**选择"是否已故"（自己不能把自己标记为已故）。

**编辑家人的资料**（从他人头像进入）：
```
头像-姓名 → 性别 → 与谁是什么关系 → 故乡/籍贯 → 代表味道（选填） → 生日 → 更多生平信息（离开的那一年，默认折叠）
```

### 3.2 谁可以编辑资料
- 家族**创建者**和**家人本人**均可编辑个人信息。

### 3.3 关系编辑规则
- 可编辑"XX 是自己的 XX 关系"。
- 仍以 **6 个关系节点**呈现。
- **仅支持单选关系**。
- 改动关系后，自动更新软件中的**家谱树**及**关系称呼**。

### 3.4 关系称呼的呈现格式
- **通用格式**：所有称呼均以"关系·名字"形式呈现。例："父亲·张三"。
- **例外（仅限"确定关系"时）**：只能从某人物（姓名）的 **6 种关系**中选择其一：
  父亲、母亲、儿子、女儿、丈夫、妻子。

### 3.5 已故状态的两种来源
- **认领前就已故**：在创建身份时即标记。
- **认领后已故**：在修改这个人的资料信息时标记。

### 3.6 家族切换（换家族）
- **位置**：味道桌页**左上角**。
- **行为**：更换家族后，**所有与家族相关的数据动态更换**。

### 3.7 ★家族权限：创建者 vs 非创建者（成员）
> 注意区分**家族创建者**与**非创建者（普通成员）**的权限，前端需按当前用户角色隐藏/禁用对应入口，后端需做二次校验（不能仅靠前端隐藏）。

**仅「创建者」可执行的操作（家族管理级）**：
- **审核请求**：通过 / 驳回 家人的加入申请、节点认领请求。
- **家族信息**：编辑家族名称 / 姓氏 / 简介。
- **家族代码**：查看、重置家族代码（重置后旧码失效）。
- **成员管理**：移除成员。
- **创建者转让**：把创建者身份转让给某位家人（转让后自己降为普通成员）。
- **退出家族**：创建者**不能直接退出**，须先「转让创建者」再退出。

**「非创建者（成员）」的权限**：
- 可**直接退出**家族。
- **不**显示/不可点击上述家族管理级操作（审核、改家族信息、重置代码、移除他人、转让）。

**创建者与成员**均可执行的操作：
- 邀请家人（分享代码 / 二维码 / 链接）。
- 添加家人到家谱、编辑资料（编辑权限另见 §3.2：创建者可编辑全部家人资料，普通成员仅可编辑**自己本人**的资料）。
- 讲味道 / 抛钩子 / 补述 / 点亮 / 时间胶囊等创作类操作。

**边界说明**：
- 一个家族**有且仅有一位**创建者；创建者角色通过「转让」转移，不可同时存在多个。
- 角色字段以服务端返回为准（成员列表中标注 `创建者` / `成员`），前端据此渲染权限。

## 四、记忆卡与味道桌

### 4.1 年份标签（硬性规则）
- 味道桌里所有美食**必须**有明确年份的标签。
- **目的**：便于时间轴的排列生成。

### 4.2 图片上传与裁剪
- **裁剪比例**：上传的照片统一裁剪为 **4 : 3**（出于美观考虑）。

**图片上传/修改入口（2 处）**：
1. **讲一种味道时**：输入框**左方**点击图片图标后上传。多次上传时，**默认保留最新一次**的图片。
2. **已点亮味道后**：在编辑页上传图片。

### 4.3 已点亮吃食页 · 右上角菜单
- **自己点亮的吃食**：点击右上角"三个点" → 显示"**编辑记忆**"。
- **别人点亮的吃食**：点击右上角"三个点" → 显示"**查看记忆**"。

## 五、时间胶囊

### 5.1 独立性（脱离味道桌）
- 一旦某段故事在"整理成一段故事"页被**设为时间胶囊**，它就：
  - **不再**出现在味道桌。
  - **只**存在于"时间胶囊"界面。
- 后续修改**只能**在"时间胶囊"界面的"编辑"按钮处进行。

### 5.2 归属与开启后的转化
- **我赠送家人的时间胶囊**：存放于"时间胶囊"界面（我侧）。
- **家人赠送给我的时间胶囊**：被开启后，**自动转化为我味道桌上一道已点亮的吃食**。

### 5.3 "送给谁" · 多选与全选联动
- 在"整理成一段故事"页设定为时间胶囊时的"送给谁"，以及**时间胶囊编辑页**的"送给谁"，均采用统一的【全选联动规则】（定义见 §1.7）。

### 5.4 到期提醒规则
- **到期日前一天**：提醒**赠送方**，给予赠送方**取消开启的机会**。
- **到期日当天**：给**被赠送方**发送通知提醒。

## 七、算法规则（需明确定义）

### 7.1 家谱同辈年龄判定
- 家族谱中**同辈之间**，自动通过**生日**确定年龄大小。
- **目的**：帮助定位兄弟姐妹的排行（兄/弟/姐/妹）。

### 7.2 时间胶囊自动启封
- 时间胶囊需**自动提取日历信息**，据此判断是否到了**自动启封**内容的时间。

### 7.3 地图 / 时间轴数据来源
- **生成地图**时：调取吃食的**地点**信息。
- **生成时间轴**时：调取吃食的**年份**信息。
