## 三、启动方式

```powershell
npx serve "C:\Users\卢雅楠\Desktop\444444"
# 浏览器打开 http://localhost:3000/一家食光.html
```

> 不建议直接双击 html（`file://` 下个别请求可能异常）。
> **多账号同屏**：开两个浏览器窗口（一个正常、一个无痕），分别登录 A 和 B，session 互相独立，可左右并排录制实时交互。
> App 刷新会回到登录页（无会话保持），切换账号很方便。

## 四、分幕脚本（建议 8–12 分钟）

| 幕 | 演示内容 | 账号 | 看点 |
|---|---|---|---|
| ① 认证 | 注册新号 → 登录 → 忘记密码（密保找回）→ 改密 | 新号 | 用户名映射邮箱、密保问题、Edge 签发临时令牌 |
| ② 家谱 + 称谓 | A 进家谱树 → 看三代结构 → 添加家人（选关系自动定性别）→ **称谓自动推导** | A | 称谓引擎、连线布局 |
| ③ 凭码加入 | A 出示码 `CH2026` → **C 输码 → P3「找到我的位置」→ 认领「陈小雨」或 relate 定位 → 进家谱** | A + C | **凭码即入伙、跨账号** |
| ④ 创建者审核 | A 进 P52 →「待审核」看到**陈志强**的申请 → 通过 / 驳回 → D 收到通知 | A（+ D） | 创建者专属权限、通知 |
| ⑤ AI 对话（主菜） | A「讲味道」→ **P30 多轮 AI 挖掘** → 「整理成故事」→ 点亮成记忆 → AI 菜谱 / 标签 / 生平 | A | agent-chat 真实 GLM、故事生成 |
| ⑥ 多账号社交 | **B 给 A 的红烧肉补述** → A 抛钩子 @ B → **B 收到「被点名」通知** → A「戳一下」催 B → 动态 Feed 实时出现 | A + B | **跨账号互动闭环、戳一下** |
| ⑦ 传承 + 越权 | 时间胶囊 → 记忆地图（扬州 / 南京）→ 纪念册（AI 文案）；**B 试改 A 的家族信息被拦** | A + B | 胶囊、地图、**RLS 安全** |

### 多账号交互是亮点，重点拍这 4 个跨账号闭环
1. **入伙**：A 给码 → C 加入（③）
2. **审核**：陈志强申请 → A 批（④）
3. **补述 + 点名**：A 发 → B 补 / 被戳（⑥）
4. **越权被拦**：B 动 A 的东西被 RLS 拒（⑦）—— 体现「安全做到了数据库层」

## 五、注意事项

- **第④幕「审核」**：当前前端「加入家族」走的是「凭码即入伙（自动通过）」，UI 里没有产生待审申请的入口。因此申请端已**预置**陈志强的真实申请——录制时只演示 A 这一端的审核动作即可，申请端口播带过。
- **第⑤幕 AI 对话**会真实调用智谱 GLM（glm-5.1 是推理模型），单轮可能有几秒延迟，属正常；录制时可适当剪辑等待。
- 家族名带 `TEST_` 前缀、登录用户名带 `test_demo_` 前缀（项目测试数据卫生纪律）；家谱里**显示的是真实中文名**，画面干净。
