# 《一家食光》页面交互构建清单（交给 Claude Design）

> 用途：让生成工具一次看清「要做哪些页面、每页展示什么、有哪些可点元素、点了去哪」。
> 共 **17 个主页面**，按用户旅程（登录 → 注册 → 入伙 → 创作 → …）排列；另有随属轻量页（搜索结果、我的记忆列表、消息中心、关于、帮助），在所属页面内一并生成。
> 记号：`「元素」→ 效果 / 跳转`；`〔弹窗〕` 为浮层，含其展示内容与按钮。
>
> **已确认的视觉基调（以实际设计稿为准）**
> - **Slogan**：食有来处，家有回声
> - **配色**：暖米色 / 米白背景 + 深墨绿主色（按钮、选中态、强调），水彩插画风
> - **弹窗形态**：所有弹窗均为从底部滑入的 bottom sheet，背景页面变暗，不使用居中弹窗
> - **家族代码**：6 位数字，输入时用 6 个独立方格，逐格填入

---

## 全局规则（所有页面通用）

- **底部导航栏**（常驻于 P10/P20/P40/P50 四个 Tab）：`「味道桌」→P10`、`「家谱」→P20`、`「家庭动态」→P40`、`「我的」→P50`；`中央「＋」→ 唤起〔添加味道入口〕`（不切换 Tab）。
- **返回**：所有非 Tab 页面左上 `「返回」→ 回上一页`。
- **家族切换的全局作用**：切换当前家族后，P10 / P20 / P40 的内容整体刷新为该家族的数据。
- **待确认态**：若加入家族需审核且尚未通过，用户可浏览，但发起记忆、补充、邀请等写操作时统一拦截并提示"待家人确认后即可参与"。
- **登录态失效** → 自动回 P1。
- **Toast**：写操作成功后给一句轻提示（如"已添加""已发送"），不阻断。

### 角色与权限（贯穿全产品）
产品只有两个角色：**创建者**与**成员**。内容层面人人权限平等，创建者仅多出家族级操作权。

| 操作 | 创建者 | 成员 |
|---|---|---|
| 编辑 / 删除任意锚点、留言、视角、复刻 | ✓ | ✓ |
| 编辑任意家谱节点资料 | ✓ | ✓ |
| 标记已故（纪念模式） | ✓ | ✓ |
| 审核认领 / 关系请求 | ✓ | ✓ |
| 移除成员 | ✓ | ✓ |
| 编辑家族信息 / 重置家族代码 | ✓ | ✓ |
| 转让创建者身份 | ✓ | ✗ |
| 解散家族 | ✓ | ✗ |

### 可见范围（统一规则）
可见范围按**每条贡献**设置（主故事、留言、视角、复刻各自独立），由贡献者本人控制，默认值取 P51"隐私默认可见"。**所有聚合视图——家庭动态、搜索结果、记忆地图、纪念册、时间胶囊——一律按查看者的权限过滤**，无权成员看不到对应条目。

### 完整度（统一口径）
"点亮数"会因新灰锚点出现而让分数式进度倒退（4/7 → 4/9），反直觉。因此**界面统一用两个数字分开表达："已点亮 X 道 · 还有 Y 道在等"**，不使用百分比或分数进度条。

### 消息与红点
认领结果、被戳一下、钩子指定给我、待审核请求等事件统一进入消息中心（入口在 P50），相关 Tab 同步亮红点；点消息条目跳转对应页面。

### 对外分享
P11 的「分享」生成只读图片卡片，**仅包含"全家可见"的内容**，受限可见的部分不会出现在分享卡中。

## P0 启动页
**展示**：全屏沉浸式水彩插画（一家人在竹林中野餐的场景）；画面底部叠加：碗形图标 + 大字「一家食光」+ slogan「食有来处，家有回声」；最底部"轻触跳过"小字。无地址栏，完全沉浸。
**交互**：
- `轻触任意位置 / 「轻触跳过」→ 进入分流逻辑`
- 若无操作，数秒后自动进入分流
**逻辑**：未登录 `→P1`；已登录且无家族 `→P2`；已登录但仅有待审核家族 `→P10（该家族「待确认」态）`；已登录有家族 `→P10`（上次所在家族）。
**〔版本更新〕**（命中新版本）：`「立即更新」→ 跳更新` ｜ `「稍后」→ 关闭并继续分流`。

## P1 登录页
**展示**（自上而下）：顶部水彩插画小图（蒸笼包子）+ 字间距拉开的「一 家 食 光」+ slogan「食有来处，家有回声」；「用户名」输入框；「密码」输入框（右侧"显示"切换）；右对齐"忘记密码？"；深墨绿「登录」大按钮；底部"还没有账号？**立即注册**"。
**交互**：
- `「登录」→ 校验`：成功 → P0 分流；失败 → 输入框下红字提示"用户名或密码错误"。
- `「立即注册」→ 〔注册 sheet〕`
- `「忘记密码？」→ 〔找回密码 sheet · Step 1〕`

**〔注册 sheet〕**（底部滑入，标题"创建新账号"）：
- 用户名：placeholder "字母、数字、下划线"，实时校验占用
- 密码：placeholder "至少6位"
- 确认密码：placeholder "再输入一次"，不一致 → 红字
- 昵称：placeholder "家人认识你的称呼"
- 密保问题区域（固定展示，不可折叠）：显示预设问题文字 + 答案输入框 "请填写答案并牢记"
- 勾选框：已阅读并同意《用户协议》与《隐私政策》
  - `「用户协议」/「隐私政策」→ 〔m-协议全文〕`
- `「完成注册」→ 所有字段填写 + 勾选协议后按钮激活 → 注册成功自动登录 → P2`
  - 〔m-协议全文〕：滚动正文；`「我已阅读」→ 关闭`。

**〔找回密码 sheet · Step 1〕**（标题"找回密码"）：
- "用户名" + 输入框 placeholder "输入你的用户名"
- `「下一步」→ 输入内容后激活 → 校验用户名存在后 → 〔找回密码 sheet · Step 2〕`

**〔找回密码 sheet · Step 2〕**（同一 sheet 内切换）：
- "请回答你设置的密保问题"
- 显示该账号的密保问题文字（如"你小时候最爱吃的一道菜叫什么？"）
- 答案输入框 placeholder "请输入答案"
- `「验证」→ 答案正确 → 〔找回密码 sheet · Step 3〕`；错误 → 红字提示；连续错误 5 次 → 锁定 24 小时

**〔找回密码 sheet · Step 3〕**：
- 新密码 + 确认新密码
- `「完成」→ 密码更新成功 → 关闭 sheet → 可用新密码登录`

## P2 入伙引导页（首次无家族）
**展示**：顶部水彩插画（家人在树下围聚的场景，占约上半屏）；右上角"退出登录"小字按钮；下方白底区域：「找到你的家族」标题 + "与家人共同守护那些流传已久的味道与记忆"副标题；「创建家族」深墨绿卡片（左侧食物插图 + 文字 + 右箭头）；「加入家族」白色卡片（左侧食物插图 + 文字 + 右箭头）；底部灰色小字"一个账号最多加入3个家族"。
**交互**：
- `「创建家族」→ 〔创建家族 sheet〕`
- `「加入家族」→ 〔加入家族 sheet〕`
- `「退出登录」→ 确认后回 P1`

**〔创建家族 sheet〕**（标题"创建家族"）：
- 家族名称：placeholder "例：陈氏大家庭、我们一家人"
- 家族姓氏：placeholder "例：陈、刘、王"（可选）
- 我的姓名：placeholder "例：陈小明"
- `「创建家族」→ 名称与姓名填写后激活 → 生成家族、本人成为家谱首节点 → 关闭 sheet → 〔m-分享家族代码〕`
  - 〔m-分享家族代码〕：显示6位代码 + 二维码 + "把代码分享给家人，邀请他们加入"；`「复制代码」→ Toast` ｜ `「分享」→ 系统分享` ｜ `「完成」→ P10（空态）`。

**〔加入家族 sheet〕**（标题"加入家族"）：
- 说明文字"向已加入家族的成员索取6位代码"
- 6个独立方格输入框，逐格填入数字，填满后自动触发校验
- 辅助输入条"输入6位代码"（点击唤起键盘）
- `「下一步」→ 6位填满后激活 → 校验`：有效 `→ 关闭 sheet → P3`；无效 `→〔m-代码无效〕`；已满3个 `→〔m-已达上限〕`。
  - 〔m-代码无效〕：`「重试」→ 清空重填`。
  - 〔m-已达上限〕：`「去管理家族」→ P52` ｜ `「关闭」`。

## P3 加入家族 · 找到你的位置
**展示**：顶部水彩插画（与 P2 同款，约占上 1/3）；「找到你的位置」标题；三步进度指示器（选择方式 → 建立关系 → 完善资料）；下方为当前步骤内容。
**交互**：

**Step 1（选择方式）**：
- `「我和某位家人有关系」（左侧握手 emoji）→ Step 2A`：说明"告诉我们你与谁相关，系统自动推导你在家谱中的位置"
- `「我就是谱上的某个人」（左侧放大镜 emoji）→ Step 2B`：说明"家人已提前添加了你，认领属于你的节点"

**Step 2A（建立关系）**：
- 成员列表（可滚动）：每条显示头像占位 + 姓名 + "TA 的味道：XX"（该成员关联的代表吃食）；尚无吃食的成员显示"待点亮…"；点某人 → 该行高亮 + 打勾
- 选中后下方出现「XX 是我的——」关系选择区：标签按钮组（父亲 / 母亲 / 儿子 / 女儿 / 丈夫 / 妻子 / 兄弟 / 姐妹 / 祖父 / 祖母……），选中标签变深墨绿
- 选定后出现预览文字："XX 是我的 XX，系统将据此推导你与家族其他成员的关系。"
- 兜底：若关系人不在谱上，`「TA 不在谱上？先把 TA 添进来」→ 填名字 + 关系 → 生成占位节点 → 继续`
- `「确认关系」→ 激活后 → Step 3`（系统在后台推导全家关系，Step 3 填完后正式写入）

**Step 2B（认领节点）**：
- "以下是尚未被认领的家族节点，找到你的名字"
- 列表：每条显示姓名 + "由 XX 添加 · 待认领" + 右侧「这是我 ›」按钮
- 若经**认领邀请深链**进入 → 跳过列表直接定位到对应节点
- `「这是我 ›」→ 进 Step 3（带入该节点信息）`
- `「我的名字不在这里？」→ 回 Step 1 选择"我和某位家人有关系"`

**Step 3（完善资料）**：
- 圆形头像上传区（相机图标 + "上传头像"）；`点击 → 拍照 / 相册`
- 你的姓名：placeholder "例：小明、二伯、外婆"；下方提示文字"将显示在家谱节点上，可随时修改"
- 生日（可选）：placeholder "例：1990年3月15日"
- `「完成，进入家族」→ 姓名填写后按钮激活 → 写入家谱 → P10`（当前家族自动切换为新加入的家族）

**〔认领待确认〕**（若该家族开启审核）：`「我知道了」→ 进 P10「待确认」态`。

## P10 味道桌（首页 · Tab）
**展示**：顶部家族切换器 + 完整度（"已点亮 4 道 · 还有 3 道在等"，口径见全局规则）+ 搜索图标；主体为竖向锚点卡片流（已点亮暖色 / 灰锚点带来源标签混排）；底部导航 + 中央＋。空态（家族暂无锚点）：可滑动"记忆线索卡" + 输入框 + 「添加一道味道」，不出现专属灰物件。
**交互**：
- `家族切换器 →〔家族切换〕`
- `完整度文字 →〔进度说明〕`
- `搜索图标 → 进入搜索态`（输入实时筛选锚点 / 家人；`点锚点结果 → P11`，`点家人结果 → P21`）
- `已点亮锚点卡片 → P11`
- `灰锚点卡片 →〔灰锚点提示〕`
- 空态：`线索卡 → P30（带该线索预填）`；`输入框 +「添加一道味道」→ P30`
- `中央「＋」→〔添加味道入口〕`

**〔家族切换〕**：列出我加入的家族（≤3，当前高亮）`→ 点某个 → 切换并刷新当前家族数据，关闭` ｜ `「加入 / 创建家族」→〔创建家族〕/〔加入家族〕`。
**〔添加味道入口〕**：`「讲一道味道」→ P30` ｜ `「采访长辈」→ P30（采访模式）` ｜ `「抛个钩子」→〔m-抛钩子〕` ｜ `「拍张照开始」→〔m-拍照来源〕`。
- 〔m-抛钩子〕：填吃食名 + 关联谱上某人 + 指定给谁（某家人 / 全家）`→「抛出」→ 在味道桌生成一个灰锚点（带"X 想听"标签），关闭 + Toast`。
- 〔m-拍照来源〕：`「拍照」→ 相机` ｜ `「相册」→ 选图 → 进 P30（带图）`。
**〔进度说明〕**：图文解释点亮机制与灰锚点三来源；`「知道了」→ 关闭`。
**〔灰锚点提示〕**："这道还没人讲过（妈妈想听），你来讲讲？" `「我来讲」→ P30（预填该线索）` ｜ `「转给家人」→ 选家人 → Toast，关闭`。

## P30 记忆挖掘 Agent · 对话流
**展示**：顶部当前吃食名（可编辑）+ 左上退出 + 右上"采访长辈"开关；中部对话区（Agent 一次一问 / 用户气泡 / 考据补充卡 / Agent 建议气泡 / 敏感话题轻提示）；底部输入栏（文字 / 语音 / 拍照）。
**交互**：
- `吃食名 → 行内编辑`
- `左上「退出」→〔退出确认〕`
- `右上「采访长辈」开关 → 切换采访模式（显示可逐条照问的问题清单 + 录长辈原声）`
- `输入栏发送 → 提交回答，Agent 继续追问`
- `「语音」→〔语音输入〕`，`「拍照」→ 拍 / 选图插入对话`
- `Agent 建议气泡（如"讲讲灶台"）→ 采纳为新线索（加入待办 / 可另起锚点）`
- `考据补充卡 →〔考据详情〕`
- 结束：`「整理成故事」→ 先弹〔关联归属〕→ 再〔生成中〕→ P31`

**〔关联归属〕**（关键，先归属再生成）：选谱上节点 ｜ `「新增家人」→〔m-新增占位家人〕`；`「确认」→ 触发〔生成中〕`。**从灰锚点进入时，归属已按钩子信息预填（可改）；从 P11「讲我的版本」进入时锚点已存在，跳过此弹窗。**
- 〔m-新增占位家人〕：填名字 + 关系 `→「添加」→ 回归属并选中`。
**〔语音输入〕**：录音 / 停止 / 重录 + 转写预览 + "保留原声"开关；`「使用」→ 插入对话，关闭`。
**〔考据详情〕**：背景资料 + 来源；`「关闭」`。
**〔生成中〕**：暖色加载动画；完成 `→ 自动跳 P31`。
**〔退出确认〕**：`「保存草稿」→ 存入"我的草稿" → 回来源页` ｜ `「放弃」→ 丢弃 → 回来源页` ｜ `「继续讲」→ 关闭`。

## P31 故事生成结果页
**展示**：生成的记忆瞬间插画预览；故事 / 家书全文；新灰锚点提示卡（"我注意到你提到了灶台，已留好位置"）；可见范围控件；底部双按钮。
**交互**：
- `插画 → 全屏预览`
- `「编辑」→〔编辑故事〕`；`选中段落「重写这一段」→ 局部重生成`
- `「重新生成插画」→〔重新生成插画〕`
- `新灰锚点提示卡「管理」→〔确认新增灰锚点〕`
- `可见范围控件（控制本篇主故事）→ 仅自己 / 部分家人（→〔m-选择可见家人〕）/ 全家`
- `「收进味道桌」→ 若由灰锚点而来，点亮**原灰锚点**（不新建重复卡片）；否则新建并点亮 → P11`
- `「继续补充」→ 回 P30（对话上下文保留，再次整理时更新本篇故事而非另起新篇）`

**〔编辑故事〕**：文本编辑；`「保存」→ 回 P31 更新`。
**〔重新生成插画〕**：`「换场景」` ｜ `「换风格」→〔m-插画风格选择〕`；`「生成」→ 刷新预览`。
- 〔m-插画风格选择〕：风格列表；`选定 → 回`。
**〔确认新增灰锚点〕**：勾选要保留的自动锚点；`「确认」→ 入味道桌`。

## P11 单锚点 · 记忆瞬间
**展示**：顶部场景插画特写 + 吃食名 + 贡献家人头像；故事正文（带朗读）；长辈原声播放条；多视角记忆卡并排；留言 / 补充时间线；菜谱区；关联的人；底部「我也来补充」。
**交互**：
- 右上：`「返回」→ 上一页`、`「可见范围」→〔可见范围设置〕（只控制**我贡献的内容**；无贡献则不显示此按钮）`、`「分享」→ 生成只读图片卡（仅含全家可见内容）→ 系统分享`、`「更多」→〔更多操作〕`
- `贡献家人头像 → 定位到 TA 的记忆卡 / P21`
- `故事旁喇叭 → 朗读播放 / 停止`
- `原声播放条 → 播放 / 暂停`
- `多视角记忆卡 → 展开该视角（含语音播放）`
- `留言语音 → 播放`；`留言照片 → 大图`
- `「复原这道味道」/「去复刻打卡」→〔菜谱详情〕`
- `关联的人 → P21`
- `底部「我也来补充」→〔补充记忆〕`

**〔补充记忆〕**：两条路径，边界清晰——
- `「留个言」（简短）→ 文字 / 语音（→〔m-语音录制〕）/ 拍照 →「留下」→ 存入留言时间线并刷新，关闭`。

- 〔m-复刻打卡〕：拍成品照 + 写一句 `→「提交」→ 锚点加深（光效 / 进度增强）+ 生成一条动态，关闭 + Toast`。
**〔可见范围设置〕**：仅自己 / 部分家人（→〔m-选择可见家人〕）/ 全家；`「保存」`。
- 〔m-选择可见家人〕：成员多选；`「确定」→ 回`。
**〔更多操作〕**：`「编辑」→ 编辑故事 / 信息` ｜ `「删除」→〔m-删除确认〕` ｜ `「举报」→ 提交，Toast`。（纪念模式**不在锚点层设置**——在家谱 P20 把某位家人标记为已故后，TA 关联的所有锚点自动转为纪念呈现。）
- 〔m-删除确认〕：若锚点含其他家人的留言 / 视角，红字警示"将同时移除家人留下的 X 条记忆"；`「确认删除」→ 删除锚点 → 回 P10` ｜ `「取消」`。
